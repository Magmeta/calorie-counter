# Calorie Counter — Документация проекта

## Описание
PWA-приложение (Progressive Web App) для подсчёта калорий с интеграцией ИИ.
Язык интерфейса: русский.

---

## Основные функции

### 1. Ввод еды
- Пользователь отправляет **фото** или **текст** с описанием того что съел
- ИИ распознаёт продукт/блюдо, считает калории и БЖУ
- Для повышения точности ИИ может задать уточняющие вопросы (вес порции, состав, дополнительные фото)
- Используются базы данных продуктов (USDA API, FatSecret API) для точных данных по БЖУ

### 2. Один чат — два режима
Переключатель вверху чата:
- **"Дневник"** — всё что пользователь пишет записывается в таблицу
- **"Вопрос"** — просто справочный вопрос, ничего не записывается

### 3. Элементы чата
- Сам чат (диалог с ИИ)
- Кнопка **"Таблицы"** → боковая панель справа (переключение дни/недели)
- Кнопка **"Мои привычки"** → список привычек пользователя (ИИ и пользователь могут добавлять/удалять)
- Кнопка **"Добавить воду"** → модальное окно с полем ввода миллилитров (только обычная вода, чай/кофе не считаются)

### 4. Уведомления в чате
- Когда ИИ запоминает привычку → уведомляет пользователя: "Я запомнил что твой чай = чай + 2 ложки сахара"
- Когда ИИ записывает данные в таблицу → короткое уведомление пользователю

### 5. Память ИИ (привычки)
ИИ запоминает важные детали чтобы не переспрашивать:
- "Чай = чай + 2 ложки сахара" → автоматически +20 ккал
- "Кофе = латте с молоком"
- "Не ест свинину" → не рекомендует
- "Аллергия на орехи" → никогда не предлагает
- Привычки добавляются двумя способами: ИИ сам из контекста разговора + пользователь вручную

### 6. Редактирование таблиц
- **Только ИИ** редактирует таблицы (пользователь не может редактировать напрямую)
- Если пользователь хочет исправить запись — просит ИИ в чате: "Убери пиццу из сегодня"
- Это предотвращает поломку данных и неправильные итоги

## Таблицы

### Дневная таблица
Колонки: Дата | Что ел | Вода (л) | Ккал | Норма Ккал | Б / Ж / У

Последняя строка — **"Итого"** (часть таблицы, разделена линией):
- Сумма ккал, воды, БЖУ за день
- Мнение ИИ в текстовом виде: недобор/перебор, рекомендации, предположения по настроению

### Недельная таблица
- Те же колонки что у дневной
- Каждая строка — один день (итог за день)
- После всех дней (через разделитель):
  - **Среднее за неделю** — средний показатель пользователя
  - **Ожидаемое** — норма/ожидаемый показатель
  - **Итог** — текстовый комментарий ИИ за неделю

### Месячная таблица
- Каждая строка — средний показатель за неделю
- В конце: среднее за месяц, ожидаемое, итог ИИ

### Каскадная группировка (боковая панель)
- **Вкладка "Дни"** — только негруппированные дни (текущая незавершённая неделя)
- Когда неделя заканчивается → дни сворачиваются в недельную таблицу
- **Вкладка "Недели"** — только негруппированные недели
- Когда месяц заканчивается → недели сворачиваются в месячную таблицу

### Настраиваемые колонки
- Базовая таблица содержит колонки описанные выше
- Колонка **"Настроение"** — доступна в настройках (5 смайликов от грустного до весёлого)
- ИИ всё равно спрашивает настроение даже если колонка скрыта, и учитывает его в итогах
- Пользователь может настроить какие колонки отображать

## Профиль пользователя

### Данные профиля
- Рост
- Вес
- Возраст
- Пол
- Уровень активности
- Цель: похудеть / набрать вес / поддерживать вес / набрать мышечную массу

### Аватар телосложения
- Визуальная фигура человека
- Меняется в зависимости от данных профиля
- Варианты: худой / средний / полный
- Рост аватара соответствует росту пользователя (маленький и полный, высокий и худой и т.д.)

### Отчёты (в профиле)
- Недельные и месячные таблицы (такие же как в боковой панели чата)

## Стек технологий

| Слой | Технология |
|------|-----------|
| Frontend | React + Next.js (TypeScript) |
| Backend | Java + Spring Boot |
| База данных | PostgreSQL |
| ИИ | Claude API или OpenAI API |
| PWA | manifest.json + Service Worker |
| Будущее (мобилка) | Capacitor |

## Технические решения
- PWA (Progressive Web App) — работает как сайт, устанавливается на телефон
- В будущем возможна обёртка в нативное приложение через Capacitor для публикации в App Store
- API баз продуктов: USDA (бесплатная), FatSecret API
- Frontend выбран React + Next.js из-за лёгкого перехода на мобилку (Capacitor, React Native)
- PostgreSQL выбран за поддержку JSON-полей, хорошие агрегации и совместимость со Spring Boot
